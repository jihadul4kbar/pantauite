## 2. ANALISA CELAH KEAMANAN

### 2.1 Level Kritis 🔴

#### SEC-01: Tidak Ada Rate Limiting untuk Login
- **File:** `app/Http/Controllers/Auth/LoginController.php`
- **Masalah:** Tidak ada throttle atau rate limiting pada login endpoint
- **PRD Reference:** 05-user-roles-permissions.md:946 (Brute Force Protection)
- **Risiko:** Vulnerable terhadap brute force attack
- **Rekomendasi:** Tambahkan rate limiting di `bootstrap/app.php` atau `RouteServiceProvider`:
  ```php
  RateLimiter::for('login', function (Request $request) {
      return Limit::perMinutes(5, 10)->by($request->email . '|' . $request->ip());
  });
  ```

#### SEC-02: Tidak Ada Security Headers
- **File:** Tidak ada middleware untuk security headers
- **Masalah:** Tidak ditemukan konfigurasi Content-Security-Policy, X-Frame-Options, X-XSS-Protection, X-Content-Type-Options
- **Risiko:** Vulnerable terhadap clickjacking, XSS attacks, MIME-type sniffing
- **Rekomendasi:** Buat middleware `SecurityHeaders` atau tambahkan di `bootstrap/app.php`:
  ```php
  $middleware->web(append: [
      \App\Http\Middleware\SecurityHeaders::class,
  ]);
  ```

#### SEC-03: File Uploads ke Public Disk Tanpa Access Control
- **File:** `config/filesystems.php:41-48`
- **Masalah:** Files disimpan di `storage/app/public` yang accessible via `/storage` symlink tanpa authorization check
- **Risiko:** Data leakage — siapapun yang tahu URL bisa download ticket attachments, asset documents, maintenance photos
- **Rekomendasi:** 
  - Pindahkan ke `local` disk (private)
  - Buat route untuk download file dengan authorization check
  - Gunakan signed URL atau streaming response dengan policy check

#### SEC-04: Potential SQL Injection pada Search Queries
- **File:** `app/Http/Controllers/RepairRequestController.php:162`
- **Masalah:** Pattern `"%{$search}%"` pada LIKE query
- **Risiko:** Meskipun Laravel Eloquent umumnya aman, pattern ini berbahaya jika ada raw query lain yang serupa
- **Rekomendasi:** Gunakan parameter binding yang proper atau Laravel's query builder yang sudah sanitized

### 2.2 Level Sedang 🟡

#### SEC-05: Password Policy Tidak Sesuai PRD
- **File:** `app/Http/Controllers/Auth/PasswordChangeController.php:27`, `app/Http/Controllers/UserController.php:101,157`
- **Masalah:** `Password::defaults()` dipanggil tapi tidak ada custom definition di `AppServiceProvider`
- **PRD Reference:** 01-prd.md:261 — require uppercase, lowercase, number, special char
- **Risiko:** Password lemah diperbolehkan (hanya default Laravel: 8 karakter tanpa complexity)
- **Rekomendasi:** Definisikan di `AppServiceProvider::boot()`:
  ```php
  Password::defaults(function () {
      return Password::min(8)
          ->mixedCase()
          ->numbers()
          ->symbols()
          ->uncompromised();
  });
  ```

#### SEC-06: Session Encryption Tidak Aktif
- **File:** `config/session.php:50`
- **Masalah:** `'encrypt' => env('SESSION_ENCRYPT', false)` — default false
- **Risiko:** Session data bisa dibaca jika storage compromised
- **Rekomendasi:** Set `SESSION_ENCRYPT=true` di `.env` untuk production

#### SEC-07: Debug Logging pada Production Code
- **File:** `app/Http/Controllers/RepairRequestController.php:50-53, 96, 101-106`
- **Masalah:** `\Log::info()` dengan detail file upload (nama file asli, size, mime type)
- **Risiko:** Information leakage, log file membengkak, potential PII exposure
- **Rekomendasi:** Hapus atau ganti dengan `\Log::debug()` yang hanya aktif di development, atau hapus sama sekali

#### SEC-08: Tidak Ada Vendor Policy
- **File:** `app/Policies/VendorPolicy.php` — TIDAK ADA
- **Masalah:** Model Vendor ada tapi tidak ada policy untuk authorization
- **Risiko:** Jika vendor CRUD ditambahkan, tidak ada protection
- **Rekomendasi:** Buat `VendorPolicy` sesuai pattern policy yang sudah ada

#### SEC-09: Ticket Rate Tanpa Rate Limiting
- **File:** `routes/web.php:138-142`
- **Masalah:** Route `tickets/{ticket}/rate` tidak ada proteksi terhadap multiple submissions
- **Risiko:** User bisa spam rating atau bot bisa manipulate rating
- **Rekomendasi:** Tambahkan middleware rate limiting atau cek apakah user sudah pernah rating

#### SEC-10: Tidak Ada Audit Logging untuk Critical Actions
- **File:** Berbagai controller
- **Masalah:** Tidak ada audit logging untuk user creation, role changes, permission changes, asset deletion
- **PRD Reference:** 01-prd.md:266 (Audit logging untuk critical actions)
- **Risiko:** Tidak ada trail untuk forensic jika terjadi security incident
- **Rekomendasi:** Implementasikan global audit logging atau tambahkan di setiap critical action

### 2.3 Level Rendah 🟢

#### SEC-11: Duplicate Route Definition
- **File:** `routes/web.php:219` dan `routes/web.php:222`
- **Masalah:** `Route::resource('users', UserController::class)` didefinisikan 2 kali
- **Risiko:** Tidak berbahaya tapi membingungkan dan bisa menyebabkan masalah di masa depan

#### SEC-12: Soft Delete pada RepairRequest
- **File:** `app/Http/Controllers/RepairRequestController.php:241`
- **Masalah:** `$repairRequest->delete()` — perlu verifikasi model menggunakan `SoftDeletes` trait
- **Risiko:** Jika tidak soft delete, data hilang permanen

#### SEC-13: Password Reset Token Table Ada Tapi Tidak Ada Flow
- **File:** Migration `password_reset_tokens` ada, tapi tidak ada controller/route untuk forgot password
- **Risiko:** User yang lupa password tidak bisa recover tanpa admin intervention
